# 需求分析

## 1. 赛题原文要求拆解

| # | 赛题原文 | 类型 | 我们的实现位置 |
|---|---|---|---|
| R1 | 输入 7 本医学教材 PDF | 数据 | `textbooks/`（已就绪） |
| R2 | 支持多格式上传（PDF/MD/TXT/DOCX） | 必做 | `POST /api/upload` |
| R3 | 为每本教材构建知识图谱 | 必做 | `src/kg/extract.py` + `src/kg/build.py` |
| R4 | 跨教材整合（去重 / 互补 / 提纯） | 必做 | `src/merge/align.py` |
| R5 | 整合后内容 ≤ 原 30%（字符数） | 必做 | `src/merge/compress.py` |
| R6 | 多轮对话 + 老师反馈迭代优化 | 必做 | `src/chat/agent.py` + 10 工具 |
| R7 | RAG 问答带引用 | 必做 | `src/rag/qa.py` + `graph_rag.py` |
| R8 | 知识图谱可视化 | 必做 | `GET /api/graph/*` + 前端 ECharts |
| R9 | 节点 schema：name/definition/category/chapter/page | 必做 | `KGNode` strict |
| R10 | 关系 schema 4 枚举 | 必做 | `prerequisite/parallel/contains/applies_to` 校验 |
| R11 | 类别 schema 7 枚举 | 必做 | 7 类强制 |
| R12 | Mermaid 架构图 | 必做 | [Agent架构说明.md](Agent架构说明.md) |
| R13 | Web SPA | 必做 | `src/web/` 静态挂载 |
| R14 | 后端 API | 必做 | FastAPI 23 路由 |
| R15 | 整合报告 | 必做 | `report/整合报告.md` |

## 2. 用户角色

- **学科老师 / 教研员**：上传教材，与 Agent 对话审核合并/拆分决策，导出整合报告。
- **学生（潜在）**：通过 RAG 问答查教材，得到带引用的答案 + 知识脉络。

## 3. 功能性需求

### 3.1 数据预处理
- F-1.1 PDF/MD/TXT/DOCX → RawDoc → chunks 流水线
- F-1.2 章节自动切分（H1 = chapter、H2 = section）
- F-1.3 公式 `$$...$$` 不切断
- F-1.4 chunk 大小 500-800 字，重叠 50-100 字

### 3.2 知识图谱
- F-2.1 LLM 抽取节点 + 边（4 关系枚举 + 7 类别枚举）
- F-2.2 节点稳定 ID（书内同名一致）
- F-2.3 单本 graph 持久化为 graphml + json

### 3.3 跨教材整合
- F-3.1 同义节点合并（lexical + 向量相似度）
- F-3.2 决策可审计（`decisions.json`）
- F-3.3 别名表（`alias_table.json`）
- F-3.4 边按 `(source,target,relation_type)` 去重，证据累积

### 3.4 内容压缩
- F-4.1 字符压缩比 ≤ 30%
- F-4.2 教学完整性自检：被保留节点的 prerequisite 不能丢
- F-4.3 fixpoint rescue 把丢失的前置节点加回

### 3.5 RAG 问答
- F-5.1 BGE 中文向量索引（每本一个 collection + 一个总 collection）
- F-5.2 Top-k 召回，cosine score
- F-5.3 答案必须引用 chunk（教材, 章节, 页码）
- F-5.4 知识库无相关时返回 "当前知识库中未找到相关信息"
- F-5.5 ⭐ GraphRAG：注入 1-hop 子图三元组到 prompt

### 3.6 多轮对话
- F-6.1 单 Agent + Function Calling
- F-6.2 10 个工具：search_kg, show_subgraph, compare_books, propose_merge/split, update_relation, add_evidence, recompress, export_report, rag_query
- F-6.3 写图操作产出 `diff_card`（before/after 子图）
- F-6.4 `diff_log.jsonl` 持久化
- F-6.5 `/api/chat/undo` 反向回滚

## 4. 非功能性需求

| 维度 | 指标 |
|---|---|
| 单次问答延迟 | < 8 秒（含 GraphRAG 检索） |
| 抽取并发 | 12 workers + resume，全量 4284 chunks ≈ 45 分钟 |
| 模型成本 | DeepSeek-Chat 全量 < ¥5 |
| 可移植 | LLM 走 OpenAI 兼容协议，env 切换 ModelScope/Qwen |
| 数据持久化 | graph json + ChromaDB persistent |
| 容错 | extract 失败的 chunk 写 `data/errors/`，整本不阻塞 |

## 5. 不做的事（明确放弃）

- ❌ 多 Agent 编排（5h 时间盒不划算）
- ❌ Neo4j（NetworkX + JSON 已够用）
- ❌ 用户登录/权限（单用户演示）
- ❌ Docker 镜像（uvicorn 直跑足以演示）
- ❌ 自定义前端组件库（直接用 ECharts/D3 + 原生 Vue3）
