# 系统设计文档

> 医学教材整合知识图谱 AI Agent
> 与赛题 `docs/赛题.md` 一一对应。

## 1. 设计目标

| 赛题要求 | 本系统实现 |
|---|---|
| 多教材输入（PDF/MD/TXT/DOCX） | `POST /api/upload` 多格式支持，`POST /api/parse` 异步流水线 |
| 知识图谱可视化 | `GET /api/graph/{merged|compact|book_id}` 提供 D3-friendly JSON；前端 ECharts 渲染 |
| 跨教材去重提纯 | `merge.align` 两阶段对齐：lexical bucket → BGE cosine ≥ 0.92 |
| 内容 ≤ 原 30% | `merge.compress` 按字符数严格控制；同时跑教学完整性 fixpoint |
| 多轮对话迭代优化 | `KnowledgeAgent` + 10 个工具 + diff_log + undo |
| 带引用 RAG 问答 | `rag.qa` (vanilla) + `rag.graph_rag` (创新点) |
| 4 种关系枚举 | `prerequisite / parallel / contains / applies_to` 强制校验 |
| 7 种类别枚举 | `核心概念/现象/过程/结构/物质/疾病/方法` 强制校验 |
| Mermaid 架构图 | [Agent架构说明.md](Agent架构说明.md) |

## 3. 数据流

```
原始 PDF
  └─► data/md/{book}.md         (PDF2MD)
        └─► data/raw/{book}.json (RawDoc + chapters[] + total_chars)
              └─► data/chunks/{book}.json (4284 chunks 全量)
                    └─► data/triples/{book}.json (LLM 抽取 nodes+edges)
                          └─► data/kg/{book}.{graphml,json} (per-book graph)
                                └─► data/kg/merged.json (跨教材整合)
                                      └─► data/kg/compact.json (压缩到 30%)

data/chunks/* ──► data/vector/ (ChromaDB BGE 索引)
                    └─► RAG 查询 + GraphRAG（注入 KG 子图）
```

## 4. KG schema（强约束）

### KGNode
```json
{
  "id": "07_病理生理学::node_3a8f1b2c",
  "name": "动作电位",
  "definition": "细胞膜在受到刺激后产生的快速、可逆的电位变化",
  "category": "核心概念",
  "book_id": "07_病理生理学",
  "chapter": "第二章 神经组织",
  "page": 32,
  "chunk_id": "07_病理生理学::ch02::s003::00012"
}
```

### KGEdge
```json
{
  "source": "07_病理生理学::node_aaaa",
  "target": "07_病理生理学::node_bbbb",
  "relation_type": "prerequisite",
  "description": "理解动作电位需先掌握静息电位",
  "book_id": "07_病理生理学",
  "chunk_id": "07_病理生理学::ch02::s003::00012"
}
```

### Merged 节点扩展字段
- `book_ids: list[str]` 出现过的教材
- `n_mentions: int` 总提及次数
- `alt_definitions: list[str]` 其他教材中的定义
- `chapters: list[str]`, `pages: list[int]`, `chunk_ids: list[str]`

## 5. 关键算法

### 5.1 两阶段对齐
1. **Lexical bucket**：按 `(normalize_name, category)` 分桶，桶内全部合并。`normalize_name` 做 NFKC + 标点剥离 + 小写。
2. **BGE 余弦匹配**：对剩余节点，同 category 内两两计算 BGE 向量余弦相似度 ≥ 0.92 才合并。

每次合并产生 `decision` 记录：
```json
{"decision_id":"dec_001","action":"merge","affected_nodes":["a::node_x","b::node_y"],
 "result_node":"merged::abc","reason":"...","confidence":0.94,"stage":"embedding"}
```

### 5.2 字符压缩
- 评分 `score(node) = 2 × n_books + log1p(n_mentions) + 0.5 × log1p(degree)`
- 按分数降序保留，直到 `compact_chars / original_chars ≤ 0.30`
- `compact_chars = Σ |definition| + Σ |Σ descriptions|`
- `original_chars = Σ data/raw/*.total_chars`

### 5.3 教学完整性 rescue
- 输入：`kept_set`、`merged_graph`
- 反复扫描：若 `B ∈ kept` 且存在 `A -[prerequisite]-> B`，则 `kept_set ∪= {A}`
- 直到不再变化
- `data/report/integrity.json` 记录 `passed` / `rescued` 数量

## 8. 性能与成本

| 阶段 | 单位 | 估算 | 成本 |
|---|---|---|---|
| PDF2MD | 7 本 | ~2 min/本 | 免费 |
| Chunker | 4284 chunks | < 30 s | 免费 |
| Extract | 4284 chunks × 1 LLM | 12 workers ≈ 25 min | DeepSeek ≈ ¥3 |
| Merge | once | < 30 s（lex）+ 1 min（BGE） | 免费（本地 BGE） |
| Compress | once | < 5 s | 免费 |
| Index | 4284 chunks | < 1 min | 免费 |
| Per Q&A | 1 query | 2~5 s | < ¥0.001 |

## 9. 演示路径

1. 上传 7 本教材 PDF → `/api/parse` 异步出 chunks/KG
2. `/api/merge/run` → 跨教材整合
3. `/api/compress/run` → 压缩到 30%
4. `/api/rag/index` → 索引向量库
5. 前端打开 → 看整合后的图谱、压缩报告
6. 进入聊天框：
   - "什么是动作电位？" → GraphRAG 出引用 + knowledge_paths
   - "动作电位 和 神经冲动 是同一个吗？" → compare_books
   - "把它们合并" → propose_merge → 推 diff_card
   - "撤销刚才那次" → /api/chat/undo
